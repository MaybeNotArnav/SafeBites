app.models.exception_model

module documentation

Defines custom exception classes used throughout the SafeBites backend. These exceptions are used to standardize error handling for API endpoints and internal services: - NotFoundException: Raised when a requested resource is not found in the database. - BadRequestException: Raised when a request is malformed or contains invalid data. - DatabaseException: Raised for database-related errors. - GenericException: Raised for generic or uncategorized exceptions. - AuthError: Raised for authentication failures (returns HTTP 401). - ConflictException: Raised for resource conflicts, such as duplicate entries (returns HTTP 409). Using these exceptions allows consistent error responses and better debugging.

Class AuthError Raised when there is an authentication error.
Class ConflictException Raised for 409 conflicts (duplicate username/dish).
Exception BadRequestException Raised when a request is malformed or contains invalid data.
Exception DatabaseException Raised when there is a database-related error.
Exception GenericException Raised for any other generic exceptions.
Exception NotFoundException Raised when a resource is not found in the database.
